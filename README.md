# pt4-population-csharp

`pt4-population-csharp` is the exploit-modeling and range-approximation engine for the Poker.RangeApprox project.

It consumes preflop population data generated from PokerTracker 4 and produces exploitative strategy recommendations by:

- approximating population ranges
- evaluating EV for all 169 starting hands
- ranking candidate hands
- constructing exploit opening and 3-bet strategies

This repository is the modeling and decision-making layer of the system.

---

## What This Repository Does

At a high level, the pipeline is:

```text
Population dataset
        ↓
Range approximation
        ↓
Population range model
        ↓
Exploit EV engine
        ↓
Exploit strategy output
```

The engine can consume data from:

- CSV exports
- PostgreSQL directly

---

## Key Features

- population range approximation
- opportunity-weighted priors
- exploit RFI analysis
- exploit 3-bet analysis
- EV decomposition by branches
- optional PostgreSQL integration
- WinForms GUI for running analyses
- ranking-based hand selection over all 169 starting hands

---

## Relationship to the Companion Repository

This project depends on population data generated by:

**`pt4-population-sql`**

That repository is responsible for:

- building materialized views from PT4
- counting opportunities and actions
- calculating frequencies
- exposing the population dataset

This repository uses those outputs to generate exploit strategies.

## Core Questions This Repository Answers

Examples:

- If the population over-folds to steals, how wide should we open?
- If the population defends too loosely, which hands become unprofitable opens?
- If a 3-bet branch is not punitive enough, which hands become good exploit candidates?
- Which hands maximize EV against the actual population rather than an equilibrium opponent?
